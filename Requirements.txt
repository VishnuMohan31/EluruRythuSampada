🏛 TRIBES INDIA MARKETPLACE PORTAL
Consolidated Requirement Document (Phase-1)

1️⃣ PROJECT OVERVIEW
Project Title
Tribes India Marketplace Portal
Objective
Develop a government-supported web application to promote and showcase tribal products under the TRIFED “Tribes India” initiative.
The platform:
	• Connects buyers and tribal vendors
	• Does NOT process payments
	• Enables communication only
	• Provides analytics for government decision-making
	• Supports configurable branding
Web application only. No mobile app.

2️⃣ CORE PRINCIPLES
	• Keep architecture simple
	• No microservices
	• No payment gateway
	• Clean UI
	• Secure and compliant
	• Government configurable
	• Scalable but minimal

your location

9️⃣ SOFT DELETE POLICY
No hard deletes.
Tables must include:
	• is_active BOOLEAN
	• deleted_at TIMESTAMP
	• deleted_by UUID (don’t use UUID. But use unique sequential increased integer id as string)
All queries must filter is_active = TRUE.

🔟 ERROR HANDLING STANDARD
All APIs return structured response:

{
  "success": false,
  "error": {
    "code": "PRD_404_001",
    "message": "Product not found",
    "details": null
  }
}
Error code format:

<MODULE>_<HTTPSTATUS>_<SEQUENCE>
Centralized exception handling required.
No stack traces in production.
All errors logged.

1️⃣3️⃣ DATABASE BACKUP & RESTORE
Backup
	• Weekly automated backup
	• pg_dump
	• Compressed (.sql.gz)
	• Stored in dedicated S3 bucket
	• Retain last 5 versions
	• Auto prune old backups
Restore
Manual restore script supported.

S3 SECURITY ISOLATION
	• Dedicated bucket
	• Private
	• EC2 IAM role restricted ONLY to this bucket
	• Application cannot access other S3 buckets
	• Bucket policy restricts to specific IAM role
	• Server-side encryption enabled
